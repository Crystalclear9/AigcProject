# 产品演示与验收脚本

这份脚本用于产品演示、内部评审和客户试用，不作为比赛叙事。重点展示“截图到行动”的真实用户闭环，而不是单点 AI 能力。

## 3 分钟核心叙事

1. 痛点：用户每天会截通知、海报、聊天承诺和报名页，但截图很快沉到相册里，真正要做的事容易漏。
2. 入口：随手办监听截图，也支持相册、系统分享和粘贴文字。只有命中明确行动证据时，才给出低打扰“可能有待办”提示。
3. 理解：端侧 ML Kit 与本地规则先生成候选卡；配置 HTTPS Workflow 网关后，后端可代理 vivo/蓝心模型、OCR 和 provider telemetry 做增强。
4. 可信：所有结果先进入候选卡，用户能逐张选择、编辑、忽略或让 AI 继续完善。
5. 落地：确认后才写入本地数据库，注册截止提醒，可选写入系统日历；确认前不产生最终副作用。

## 推荐演示顺序

1. 打开 App，确认设置页显示“手机独立运行”或“AI 增强已配置”。
2. 截一张广告/系统页/随手办自身页面，确认不会出现待办提示。
3. 截一张课程通知或从相册导入复杂课程截图，展示“可能有待办”低打扰提示。
4. 点击生成，查看候选卡里的标题、截止时间、地点/平台、材料/提交方式、证据摘要和置信度。
5. 导入多任务公告，确认拆出多张候选卡，并分别演示“全部创建”和“选择性创建”。
6. 选中一张卡，点击“让 AI 继续完善”，展示建议只影响选中卡，用户锁定字段不被覆盖。
7. 点击确认创建，进入卡片中心，展示已保存卡片、提醒注册结果和可选日历同步结果。

## 常见追问

- 为什么不是普通 OCR？
  - OCR 只拿到文字。随手办进一步完成噪声过滤、行动判定、事项拆分、字段抽取、提醒策略和确认闭环。
- 为什么不是普通待办？
  - 普通待办依赖手动录入。随手办从截图和聊天里提取候选事项，让用户确认后再落地。
- AI 错了怎么办？
  - AI 不能直接保存或提醒；所有结果先进入候选卡，低置信度字段会提示复核，用户确认前没有最终副作用。
- 没有模型 key 怎么跑？
  - 端侧 ML Kit + 本地规则可以完整完成截图判定、候选卡、保存和提醒；模型只是增强，不是单点依赖。
- 怎么证明不是写死？
  - 可换任意复杂通知文本或截图；后端 `/api/providers/status` 和 `/api/providers/probe` 可脱敏证明 provider 实际参与。
- 隐私怎么处理？
  - Android 不保存 provider key，不直连 `api-ai.vivo.com.cn`；手机只配置随手办 HTTPS Workflow 网关。用户确认前不写最终卡片、提醒或日历。

## 验收关注点

- 少打扰：无行动截图不提示，同一截图忽略后短时间不重复提示。
- 具体性：候选卡不能退化为“相关日程”，必须尽量给出标题、时间、地点/平台和材料。
- 多任务：一张公告里多个事项要拆成多张候选卡。
- 可靠性：模型、OCR、网络或权限失败时，必须有降级和可理解错误。
- 安全性：真实密钥只放后端环境变量，不进入 Android、README、脚本、APK 或日志。
