# 随手办全国赛演示脚本

## 3 分钟核心叙事

1. 痛点：大学生每天截很多通知、海报和聊天承诺，但截图躺在相册里，真正要做的事容易漏。
2. 入口：随手办监听截图或接收相册/系统分享图片，截图后用通知轻量提示“生成行动卡”。
3. AI：端侧 OCR 先提取文字，后端蓝心大模型或本地规则把文字拆成任务卡、事件卡、承诺卡、资料卡。
4. 可信：所有动作先进入预览，用户能编辑标题、时间、地点、提交方式和提醒策略。
5. 落地：确认后写入本地数据库，注册通知提醒，可进入卡片中心和日历时间线管理。

## 推荐现场演示顺序

1. 打开后端 `http://127.0.0.1:8000/docs`。
2. 调用 `GET /api/v1/demo/evaluate`，展示五类策划案场景的通过情况。
3. 打开 Android App，进入“导入”，选择“会议准备”示例。
4. 展示动作预览里自动拆出“参加组会”和“准备进展汇报”两张卡。
5. 修改一个待确认字段，点击确认创建。
6. 在“今日”展示 AI 行动闭环看板，在“卡片中心”筛选卡片，在“日历视图”展示时间线。

## 评委可能追问

- 为什么不是普通 OCR？
  - OCR 只拿到文字，随手办进一步完成语义理解、字段抽取、行动拆解、提醒策略和管理闭环。
- 为什么不是普通待办？
  - 普通待办依赖手动录入，随手办以截图为入口，覆盖课程群、比赛海报、聊天承诺等非结构化高频场景。
- AI 错了怎么办？
  - 动作预览和待确认字段是核心机制，AI 不直接执行，用户确认后才创建提醒和日历。
- 没有模型 key 怎么演示？
  - 后端和 Android 都有规则兜底，保证比赛现场能跑通；配置蓝心 key 后自动走云端模型。
- 隐私怎么处理？
  - 用户主动触发识别，设置中心提供隐私脱敏、原始截图保存策略；后端只接收 OCR 文本，默认不上传图片。

## 一等奖取向补强点

- 展示“截图到行动”的闭环速度，而不是单点 AI 能力。
- 强调端云协同和无 key 兜底的工程韧性。
- 强调待确认机制解决 AI 幻觉和产品信任。
- 用五类真实校园场景证明场景选择准确。
